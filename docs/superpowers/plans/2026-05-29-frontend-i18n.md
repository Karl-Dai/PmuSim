# Frontend i18n (中/English switching) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a runtime 中文/English toggle to the PmuSim desktop frontend, translating all user-facing UI text and frontend-mapped semantic labels.

**Architecture:** A dependency-free i18n module under `frontend/src/i18n/` — `detect.ts` (pure, Vue-free, node-testable) holds locale detection + `{var}` interpolation; `messages.ts` holds flat dot-keyed `zh`/`en` catalogs; `index.ts` wraps a reactive `locale` ref (initialised from `localStorage` or `navigator.language`), exposes `setLocale`, a standalone `t()`, and a `useI18n()` composable. Components and `usePmuEvents.ts` replace hardcoded Chinese with `t('key')`. A `中 / EN` toggle lives in the title bar.

**Tech Stack:** Vue 3.5 `<script setup>` + TypeScript, Vite 6, Tauri 2. No new runtime dependency. Pure-function tests run via `node --experimental-strip-types`. Final UI verification via Playwright (headless).

**Spec:** `docs/superpowers/specs/2026-05-29-frontend-i18n-design.md`

**Scope notes (do NOT translate):** code comments; `payload.error` strings emitted by the Rust backend (passed through verbatim in `usePmuEvents.ts`); the `useCommLog` raw-frame stream; the product token `simpmufep` (preserved verbatim in both locales — only the descriptor after it is translated). `useFrameRate.ts`, `useSessions.ts`, `useEventLog.ts` contain Chinese **only in comments** — leave untouched.

All commits: author is the repo default; **no** generated `Co-Authored-By` / signature lines.

```ts
// Pure, Vue-free i18n helpers. Unit-tested standalone via
// `node --experimental-strip-types` (no test-runner dependency).

export type Locale = 'zh' | 'en'

/** First-run resolution: an explicit stored choice wins; otherwise infer
 *  from the OS/webview locale (zh* → zh, everything else → en). */
export function detectLocale(navLang: string | undefined, stored: string | null): Locale {
  if (stored === 'zh' || stored === 'en') return stored
  return (navLang ?? '').toLowerCase().startsWith('zh') ? 'zh' : 'en'
}

/** Look up `key` in a single-locale catalog and interpolate `{var}`
 *  placeholders. Missing key → returns the key (and warns in dev). */
export function translate(
  catalog: Record<string, string>,
  key: string,
  params?: Record<string, string | number>,
): string {
  const raw = catalog[key]
  if (raw === undefined) {
    if (import.meta.env?.DEV) console.warn(`[i18n] missing key: ${key}`)
    return key
  }
  if (!params) return raw
  return raw.replace(/\{(\w+)\}/g, (_, name) =>
    name in params ? String(params[name]) : `{${name}}`,
  )
}
```

- [ ] **Step 4: Write `index.ts` (reactive wrapper)**

Create `frontend/src/i18n/index.ts`:
```ts
import { computed, ref } from 'vue'
import { detectLocale, translate, type Locale } from './detect'
import { messages } from './messages'

export type { Locale }

const STORAGE_KEY = 'pmusim.locale'

const locale = ref<Locale>(
  detectLocale(
    typeof navigator !== 'undefined' ? navigator.language : undefined,
    typeof localStorage !== 'undefined' ? localStorage.getItem(STORAGE_KEY) : null,
  ),
)

export function setLocale(l: Locale): void {
  locale.value = l
  try { localStorage.setItem(STORAGE_KEY, l) } catch { /* private mode etc. */ }
}

/** Standalone translator — reads the reactive locale, so it is reactive in
 *  templates/computed AND usable from non-component modules (composables). */
export function t(key: string, params?: Record<string, string | number>): string {
  return translate(messages[locale.value], key, params)
}

export function useI18n() {
  return { locale: computed(() => locale.value), setLocale, t }
}
```
